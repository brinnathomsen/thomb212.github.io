Process

The Goal

  1. Create a webpage that acts as a poem generator
  2. Use chance to create a changing poem
  3. Use event states to make the webpage change over time

Objectives

  • generate a poem that makes sense
  • be interactive
  • affect the user emotionally or intellectually through the interaction
  • be visually interesting & beautiful

Brainstorming a Concept

  • Change the mood of the poem based on the users interactions with it, like a person*
  • Alliterative poem that the user can control by clicking on individual letters to change the first
  • Whole page is filled with poem (but it's so small it forms more of a pattern) but the user hovers over areas to magnify them and enjoy only small parts of it at a time.
  • Generated newspaper/site

Developing the Concept

For this project, I've decided to create a webpage that generates a note to the user based on how the user is interacting with the site. I will portray the site's moods as if it was a human, hopefully making the user reflect on the 'relationship' that we have with the internet.

  • The first question to consider is what would a webpage 'feel' about different interactions with it?
  • What would make a webpage happy, sad, angry, excited?

    • The site will seem 'happy' when...

    The site will seem 'excited' when...

    The site will seem 'sad' when...

  • Since the internet is meant to help us, I can only assume that a webpage would be annoyed or sad if we ignored it

    • The site will seem 'angry/hostile' when...
